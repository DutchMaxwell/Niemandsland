================================================================================
BENÖTIGT: Lese-Endpoint für Spielzustand
================================================================================

Ein einfacher PHP-Endpoint der den Inhalt einer Spielzustand-Datei zurückgibt.


MINIMAL-IMPLEMENTIERUNG (GetGameState.php):
-------------------------------------------

    <?php
    // GetGameState.php
    header('Content-Type: text/plain; charset=utf-8');
    header('Access-Control-Allow-Origin: *');

    $gameID = $_GET['gameID'] ?? '';

    // Validierung: Nur Zahlen erlauben
    if (!preg_match('/^[0-9]+$/', $gameID)) {
        http_response_code(400);
        exit('Invalid gameID');
    }

    $file = $gameID . ".txt";
    if (!file_exists($file)) {
        http_response_code(404);
        exit('Game not found');
    }

    echo file_get_contents($file);
    ?>


AUFRUF:
-------
    GET https://udos3dworld.com/WargamingSimulator/GetGameState.php?gameID=20251102092403607291


================================================================================
OPTIONAL: JSON-Format
================================================================================

Falls strukturierte Daten bevorzugt werden:

    <?php
    header('Content-Type: application/json');
    header('Access-Control-Allow-Origin: *');

    $gameID = $_GET['gameID'] ?? '';

    if (!preg_match('/^[0-9]+$/', $gameID)) {
        echo json_encode(['error' => 'Invalid gameID']);
        exit;
    }

    $file = $gameID . ".txt";
    if (!file_exists($file)) {
        echo json_encode(['error' => 'Game not found']);
        exit;
    }

    echo json_encode([
        'gameID' => $gameID,
        'data' => file_get_contents($file)
    ]);
    ?>


================================================================================
OPTIONAL: Log-Datei mitlesen
================================================================================

Für Chat/Spielprotokoll-Synchronisation:

    $logFile = $gameID . "_log.txt";
    if (file_exists($logFile)) {
        $response['log'] = file_get_contents($logFile);
    }


================================================================================
WICHTIG: CORS-Header
================================================================================

Der Header "Access-Control-Allow-Origin: *" ist notwendig, damit Niemandsland
(als Desktop-Anwendung) auf die Daten zugreifen kann.

================================================================================
RÜCKKANAL (Züge von Niemandsland → WGS)
================================================================================

Für bidirektionale Synchronisation könnte Niemandsland POST-Requests an DoAction.php
senden. Das bestehende Format (ActionString) ist bereits dokumentiert und
verstanden. Hierfür wäre ebenfalls CORS notwendig.
