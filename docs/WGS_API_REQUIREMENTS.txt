================================================================================
Niemandsland ↔ WGS Integration - Technical Requirements
================================================================================

CURRENT STATE
-------------
Niemandsland already has a WGS integration implemented:
- Parser for the WGS game-state format ({gameID}.txt)
- Conversion of the coordinates (inches → meters)
- Spawning of units based on WGS data
- Support for multibase units, colors, rotation
- Parsing of the OPR stats from the name field (Q+/D+, weapons, points)


STATUS: API ENDPOINT ONLINE ✓
-----------------------------
The getGameState.php endpoint is now available:

    https://udos3dworld.com/WargamingSimulator/getGameState.php?gameID={gameID}

Niemandsland uses this endpoint for the server import.


================================================================================
REQUIRED: Read endpoint for game state
================================================================================

A simple PHP endpoint that returns the contents of a game-state file.


MINIMAL IMPLEMENTATION (GetGameState.php):
-------------------------------------------

    <?php
    // GetGameState.php
    header('Content-Type: text/plain; charset=utf-8');
    header('Access-Control-Allow-Origin: *');

    $gameID = $_GET['gameID'] ?? '';

    // Validierung: Nur Zahlen erlauben
    if (!preg_match('/^[0-9]+$/', $gameID)) {
        http_response_code(400);
        exit('Invalid gameID');
    }

    $file = $gameID . ".txt";
    if (!file_exists($file)) {
        http_response_code(404);
        exit('Game not found');
    }

    echo file_get_contents($file);
    ?>


CALL:
-------
    GET https://udos3dworld.com/WargamingSimulator/GetGameState.php?gameID=20251102092403607291


================================================================================
OPTIONAL: JSON format
================================================================================

If structured data is preferred:

    <?php
    header('Content-Type: application/json');
    header('Access-Control-Allow-Origin: *');

    $gameID = $_GET['gameID'] ?? '';

    if (!preg_match('/^[0-9]+$/', $gameID)) {
        echo json_encode(['error' => 'Invalid gameID']);
        exit;
    }

    $file = $gameID . ".txt";
    if (!file_exists($file)) {
        echo json_encode(['error' => 'Game not found']);
        exit;
    }

    echo json_encode([
        'gameID' => $gameID,
        'data' => file_get_contents($file)
    ]);
    ?>


================================================================================
OPTIONAL: Also read the log file
================================================================================

For chat/game-log synchronization:

    $logFile = $gameID . "_log.txt";
    if (file_exists($logFile)) {
        $response['log'] = file_get_contents($logFile);
    }


================================================================================
IMPORTANT: CORS header
================================================================================

The header "Access-Control-Allow-Origin: *" is necessary so that Niemandsland
(as a desktop application) can access the data.
